# Repository example — clean-panel

**Everything in this example is fictional.** It demonstrates the
scholar-reach workflow on an early-stage open-source research tool —
the case where maturity inflation ("production-ready", "widely
adopted", invented stars) is the main risk.

## The fictional artifact

- Name: **clean-panel**
- Purpose: a reproducible workflow for auditing and cleaning
  longitudinal survey data
- Status: **early-stage prototype**

## What this example demonstrates

1. Maturity is stated honestly: every content item says early-stage
   prototype; nothing claims production readiness or wide adoption.
2. No adoption metrics appear anywhere — none exist, so none are
   mentioned (see the absence of `adoption_metric` rows in the ledger).
3. Capabilities vs. design goals are typed separately: what the tool
   does today (`software_capability`) vs. what it aims to do
   (`software_design_goal`) — see
   [`output/03_claims/claim_ledger.csv`](output/03_claims/claim_ledger.csv).
